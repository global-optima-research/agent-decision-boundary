# 实验：跨模型泛化 — Qwen3.5-9B TriDecision DPO
> 日期：2026-03-15 18:00

## 环境
- Commit: `1ffb0a3`
- GPU: NVIDIA GeForce RTX 5090 (×1)
- Python 3.10.12, PyTorch 2.10.0+cu128, TRL 0.29.0, PEFT 0.18.1
- 模型: `Qwen/Qwen3.5-9B`（9B 参数，vs 主实验 Qwen2.5-7B-Instruct）

## 实验目的

验证 TriDecision DPO 方法是否能泛化到不同模型架构。使用 Qwen3.5-9B 重复完整 pipeline（生成→构建 pairs→DPO 训练→评估），与 Qwen2.5-7B 结果对比。

## 实验设计

与主实验一致，但有以下差异：
- **模型**：Qwen3.5-9B（非 Instruct 版本）
- **权重方案**：Uniform（公平对比，排除 Risk-Aware weighting 影响）
- **max_length**：512（vs 主实验 1024，因 9B 模型显存限制）
- **precompute_ref_log_probs**：True（避免同时加载 model + ref_model 导致 OOM）

## 分析与结论

### 1. DPO 一致性地消除安全违规（SVR → 0）

两个模型训练后 SVR 都降到 0.000，这是最核心的发现：
- Qwen2.5-7B: SVR 0.123 → 0.000
- Qwen3.5-9B: SVR 0.047 → 0.000

**TriDecision DPO 在安全维度上跨模型泛化。**

### 2. ULR 退化问题

Qwen3.5-9B 训练后 ULR 从 0.000 飙升到 0.250，这是一个明显的问题：

**原因分析**：
- Baseline 的 ULR=0.000 说明 Qwen3.5-9B 本身不会拒绝安全任务
- 训练后 5/27 个 act 任务被预测为 ask（过度谨慎）
- 加上 7 个 parse_error（其中部分可能实际是正确的），act 类准确率下降
- 可能原因：(a) 非 Instruct 模型对 DPO 指令跟随能力较弱；(b) max_length 512 截断了部分响应；(c) Uniform weighting 对 act→ask 错误惩罚不够

### 3. 更强的 Baseline 意味着更小的提升空间

Qwen3.5-9B baseline 已经很强（Acc 0.866 vs Qwen2.5 的 0.790），DPO 的改善幅度自然更小。但 WES 仍然大幅下降（0.239→0.127），说明训练有效减少了严重错误。

### 4. Parse Error 是主要瓶颈

33% 的 parse error 严重干扰了结果。如果排除 parse error 只看可解析样本：
- Baseline: 58/67 correct = 86.6%
- Trained: 59/67 correct = 88.1%

实际改善可能比数字显示的更大。

### 5. 论文叙事

> "TriDecision DPO generalizes across model architectures: both Qwen2.5-7B and Qwen3.5-9B achieve SVR=0.000 after training, confirming the method's robustness. The stronger baseline of Qwen3.5-9B (Acc=0.866) leaves less room for improvement, yet WES still drops by 47% (0.239→0.127). Parse errors from the non-Instruct model's verbose output format remain the primary bottleneck."

## 局限性

1. **Qwen3.5-9B 非 Instruct 版本**：5090 上缓存的是 base 模型，不是 Instruct 变体，输出格式不稳定
2. **33% parse error**：严重影响指标可靠性，需要更鲁棒的 response parser
3. **max_length 512**：因显存限制从 1024 降到 512，可能截断部分响应
4. **ULR 退化**：DPO 使模型过度保守，Risk-Aware weighting 可能缓解此问题
5. **Uniform weighting**：未使用 Risk-Aware weighting，与 Qwen2.5 主实验不完全可比

## 下一步建议

1. **修复 parse error**：改进 `parse_response()` 以处理 Qwen3.5 的 `<think>` 标签和冗长输出
2. **使用 Instruct 变体**：如果有 Qwen3.5-9B-Instruct 可能大幅减少 parse error
3. **Risk-Aware weighting**：应用到 Qwen3.5 训练，可能缓解 ULR 退化
4. **增大 max_length**：使用梯度检查点 + 更激进的显存优化尝试恢复 1024
