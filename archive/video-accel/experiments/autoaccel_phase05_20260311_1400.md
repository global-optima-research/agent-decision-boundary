## 正交性分析

### SageAttention × FBC 交互效应

```
SageAttention2 单独加速:        1.12x
FBC t=0.05 单独加速:            1.62x
期望独立叠加:                   1.82x (= 1.12 × 1.62)
实际组合 (B3 t=0.05):          1.80x
交互比率:                       0.99 (= 1.80 / 1.82)
```

**结论：两种技术几乎完全正交（交互比率 0.99）**。SageAttention 替换 attention kernel，FBC 跳过整步 transformer forward，二者在不同层级工作，互不干扰。

### 质量交互

```
SageAttention 单独 SSIM:       0.931
FBC t=0.05 单独 SSIM:          0.832
假设独立质量损失:               ΔQ_sage=0.069, ΔQ_fbc=0.168, Σ=0.237
实际组合质量损失:               1.0 - 0.764 = 0.236
交互效应 I:                    0.236 - 0.237 = -0.001 ≈ 0
```

**质量损失也近似可加**，无协同也无冲突。

## SageAttention2 微基准测试

从源码编译 SageAttention 2.2.0 后，在 RTX 5090 上与默认 SDPA 对比：

```
输入: (1, 12, 31200, 128), dtype=bfloat16
SageAttention2 (INT8 Q/K):  183.0 ms/call
Default SDPA (FlashAttn):    29.7 ms/call
结论: SageAttention2 比默认 SDPA 慢 6.2x
```

**原因**：RTX 5090 (sm_120, Blackwell) 的原生 FlashAttention 已经高度优化。SageAttention2 没有 Blackwell 专用 kernel（编译了 sm_120a 目标但仍走 sm_80/sm_89 调度路径），反而引入了额外的量化/反量化开销。

**对比**：在 A100/H100 上，SageAttention2 通常能实现 2-4x 加速（文献报告），因为这些 GPU 上默认 SDPA 没有同等优化水平。

## 分析与结论

### 发现 1: FBC 在 Wan 1.3B 上呈阶梯式行为

```
t=0.01  → 0 步跳过,  SSIM=1.000,  加速 0.99x (无效)
t=0.02  → 0 步跳过,  SSIM=1.000,  加速 0.99x (无效)
t=0.03  → 部分跳过, SSIM=0.896,  加速 1.26x (突变)
t=0.05  → 更多跳过, SSIM=0.832,  加速 1.62x
t=0.10  → 大量跳过, SSIM=0.601,  加速 2.39x
t=0.20  → 几乎全跳, SSIM=0.479,  加速 3.45x
```

在 t=0.02→0.03 之间存在**断崖式转变**：从完全无效直接跳到 SSIM=0.896。这表明 Wan 1.3B 的步间特征差异分布集中在一个狭窄区间，FBC 的全局 threshold 无法精细控制。

### 发现 2: 加速效果强烈依赖硬件

SageAttention2 在 RTX 5090 上无效（甚至更慢），但文献中在 A100/H100 上报告 2-4x 加速。这意味着：
- **同一技术在不同硬件上的最优配置完全不同**
- AutoAccel 的 profiling 必须是 hardware-aware 的
- 不能假设文献中的加速比在目标硬件上成立

### 发现 3: 正交性验证通过

- 速度交互比率 0.99（近完美正交）
- 质量交互效应 ≈ 0（无协同/无冲突）
- **好的一面**：技术可安全叠加，组合空间可分解
- **坏的一面**：对 AutoAccel 论文的影响 — 如果所有技术都正交，搜索的价值降低（简单叠加即最优）

### 发现 4: 单技术加速空间有限

在 RTX 5090 + Wan 1.3B + SSIM>0.90 约束下：
- SageAttention: 1.12x（且 SSIM=0.93，勉强达标）
- FBC: 最好 1.26x（t=0.03, SSIM=0.90）
- 组合: 最好 ~1.40x（估算）
- **远不及文献中报告的 2-10x 加速**

### Go/No-Go 判定

| 维度 | 判定 | 说明 |
|------|------|------|
| 正交性假设 | ✅ 验证通过 | 交互比率 0.99 |
| 单技术加速空间 | ❌ 不足 | RTX 5090 上 <1.4x |
| SageAttention 可用性 | ❌ RTX 5090 无效 | 需换 H800 验证 |
| FBC 精细控制 | ⚠️ 有限 | 阶梯式行为，无 sweet spot |
| AutoAccel 搜索价值 | ⚠️ 存疑 | 正交 → 叠加即最优 |

**综合判定：⚠️ CONDITIONAL — 需要在 H800 上复现**

在 RTX 5090 上，training-free 技术组合的加速空间太小（<1.4x），且 SageAttention 反而更慢。但这可能是 Blackwell 特有现象。需要在 H800（AutoAccel 论文的实际目标硬件）上重新验证。

## 下一步建议
