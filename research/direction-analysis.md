# 研究方向战略分析

> 日期：2026-03-09
> 背景：Phase 0/1 完成后，重新评估最优研究方向
> 资源：8×H800 (训练) + 1×RTX 5090 (实验)
> 目标：顶会论文（NeurIPS 2026 / ICML 2027）

## 核心判断

**注意力加速（纯推理 kernel）方向不推荐继续**，原因：
1. 竞争极度饱和 — ICML/NeurIPS 2025 已有 6+ 篇（STA、FPSAttention、VMonarch、Radial Attention、SVG、LiteAttention）
2. "编译"角度（per-head sparse + offline profiling + mask reuse）可能被审稿人视为工程优化
3. Kernel 开发要和 Tri Dao 组竞争，风险高

**有 8×H800 训练资源后，训练型方向变得可行**，但传统步数蒸馏仍不推荐（大厂主导）。

## 推荐方向排序

### ★★★ 方向 1：加速感知蒸馏（Acceleration-Aware Distillation）

**一句话**：不是蒸馏来减步数，而是蒸馏让模型适应稀疏/缓存，使推理加速技术的质量损失趋近于零。

**动机**：
Phase 1 证明了核心矛盾 —
- 稀疏 mask 跨步很稳定（IoU 0.86），说明"哪些位置重要"是可预测的
- 但直接跳步/跳层会崩（SSIM < 0.55），说明模型没被训练来容忍近似计算

```
现有范式:  Train(full) → Infer(sparse) → 质量损失
我们的范式: Train(full + sparse-aware) → Infer(sparse) → 几乎无损
```

**做法**：
1. 用 Phase 0 数据确定每个 (layer, head) 的稀疏模式
2. 微调时随机 dropout 注意力位置（模拟推理时的稀疏）
3. 蒸馏损失 = L_task + λ·KL(full_attn_output || sparse_attn_output)
4. 推理时直接用 top-5% 稀疏，质量几乎无损

**为什么最优**：
- Phase 0/1 数据是独特壁垒，别人没有
- LoRA 微调，8×H800 迭代快（1.3B 用 2 卡，14B 用 8 卡）
- Story 清晰："模型不是不能加速，是没被教过怎么被加速"
- 竞争空白 — 没有人从这个反向角度做过
- 可叠加 — 微调后可同时用稀疏 + 量化 + 缓存

**竞品区分**：
| 工作 | 做了什么 | 我们的差异 |
|------|---------|-----------|
| Video-BLADE | 同时设计稀疏模式+蒸馏 | 我们让模型适应已知的最优稀疏模式 |
| QuantSparse | 研究量化+稀疏干扰 | 我们训练来消除干扰 |
| STA/SVG | 纯推理时稀疏 | 我们加训练环节使稀疏无损 |
| DisCa | 蒸馏学缓存策略 | 我们让特征更 cacheable |

### ★★★ 方向 2：少步蒸馏 + 时间一致性专项优化

**一句话**：现有蒸馏从图像搬到视频后时间一致性容易崩，专门设计时间一致性感知的蒸馏 loss。

**动机**：
- DOLLAR、DCM 做了视频蒸馏，但 temporal flickering 是公认未解决问题
- 图像 CM/CT loss 不含时间维度约束
- 无人专门设计 temporal-consistency-aware 蒸馏 loss

**做法**：
- 基于 Flow Matching（Wan 框架），加入时间一致性正则
- 光流一致性 loss、相邻帧 latent 相似度约束、temporal attention 保持约束
- 目标：4 步生成 + 时间一致性 > 现有 4 步方法

**8 卡可行性**：Wan 1.3B 2-4 卡几天，14B 8 卡 1-2 周

**风险**：蒸馏赛道本身拥挤，必须在时间一致性上有明确定量提升才有竞争力

### ★★★ 方向 3：组合优化框架（不需训练）

**一句话**：没人系统研究 sparse × quant × cache 的交互效应和最优组合。

**做法**：
- 单技术 profiling → 交互效应分析 → 自动搜索最优组合
- 用现有工具（SageAttention、TeaCache、STA）做组合实验
- Phase 0/1 数据直接复用

**优点**：空白度高、不需训练、1×5090 足够
**适合投稿**：MLSys 2027 / ICML 2027

### ★★ 方向 4：分析型论文

用 Phase 0/1 的反直觉发现写分析论文：
- 层位置 > 变化量（r=0.77），变化小的层跳过反而最致命
- U 形稳定性假设不成立（PAB 的前提在 Wan 上失效）
- 步间输出复用彻底失败（SSIM < 0.55）
- top-1% 覆盖 57.8%（远超文献）

**优点**：数据已有，1-2 月可投
**风险**：需扩展到更多模型增加说服力

## 不推荐的方向

| 方向 | 原因 |
|------|------|
| 纯注意力 kernel 开发 | 6+ 顶会竞品，要和 Tri Dao 组竞争 kernel |
| 传统步数蒸馏 | 大厂主导（NVIDIA FastGen、TurboDiffusion），8 卡不够暴力搜超参 |
| 纯量化 | 拥挤且依赖硬件 |
| 测试时计算×加速 | 空白但问题定义不成熟，风险高 |

## 建议执行计划

| 阶段 | 内容 | 时间 |
|------|------|------|
| 先完成 Phase 2 | 验证稀疏注意力质量（为方向 1 提供基础） | 1 周 |
| 方向 1 启动 | 设计 sparse-aware 微调方案，1.3B 上验证 | 2-3 周 |
| 扩展到 14B | 8×H800 训练，多模型评估 | 2 周 |
| 论文写作 | 投 NeurIPS 2026 (DDL ~May 2026) | 2 周 |
